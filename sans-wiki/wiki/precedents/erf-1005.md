# ERF 1005 — Illegal Second Dwelling with Structural Concerns

## Property Details
- **ERF Size:** 980m²
- **Existing improvements:** Single dwelling (approved 2008)
- **Proposed works:** 95m² "staff quarters" — effectively a second dwelling

## Departments Reviewed
| Dept | Status | Notes |
|------|--------|-------|
| BC | ❌ REJECTED | Non-compliant — exceeds 80m² second dwelling limit; structural concerns |
| RSP | ❌ REJECTED | Second dwelling not permissible at this size |

## Key Decision Points

### Second Dwelling Size Violation (Tshwane Scheme Clause 20)
- Proposed "staff quarters": 95m²
- Maximum permitted for second dwelling: 80m²
- **Exceedance: 15m²** — not a minor variation, this is a different use
- At 95m², this is functionally a second residential unit
- Would require consent use application, not a building plan

### Structural Concerns (SANS 10400-J)
- Submitted structural drawings showed:
  - Foundation on expansive clay soil — no geotech report
  - Single-skin masonry for external walls — below minimum for thermal/structural
  - No damp-proof course details
- BC department flagged all three issues

### Coverage (Clause H2)
- Proposed addition (95m²) would push coverage to 58% ✅ (just within limit)
- Not the reason for rejection but noted

### Zoning Non-Compliance
- R1 zone permits one dwelling unit per erf
- 95m² "staff quarters" constitutes a second dwelling unit
- Consent use process required — not a standard building plan

## Decision
**REJECTED** on grounds of:
1. Proposed structure exceeds 80m² second dwelling limit — requires consent use application
2. Structural design non-compliant with SANS 10400-J (masonry strength, foundation)
3. No geotechnical report for expansive soil conditions
4. Applicant must resubmit as consent use application if they wish to proceed

## Lessons Learned
- "Staff quarters" naming doesn't override the functional reality of a second dwelling
- 80m² is a hard boundary — anything over requires different approval track
- Expansive soil (common in Pretoria area) always needs geotech report
- Don't assume the application is what the applicant calls it — examine the functional reality
